Two modes: route wear simulation and shortest paths using Dijkstra.

Overview

This project contains two separate graph tasks inside the same program, selected by a command-line argument (1 or 2). I used adjacency lists to represent an undirected graph and kept the logic split by mode: one part simulates how wear spreads across route segments over the years, and the other finds the shortest paths from a start city using Dijkstra and outputs the requested edges.

C Graphs Dijkstra Adjacency Lists

Mode 1: Route wear simulation

  • Reads routes and segment wear values (floats with two decimals).
  • Simulates wear changes for K years using the rules from the statement.
  • Updates are applied step by step (not reusing newly updated values inside the same year).
  • At the end, it computes average wear per route and prints only routes under a threshold.

Mode 2: Shortest paths

  • Runs Dijkstra from the start node using arrays for distances and visited.
  • Keeps predecessor info to reconstruct which edges are part of the shortest paths.
  • Outputs the chosen edges in the input order, following the constraints.

Short explanation

The simulation runs in discrete years. For each route segment, the code updates the wear value based on the rules stated (growth for already affected segments, propagation for zero-wear segments), limited to 100.
