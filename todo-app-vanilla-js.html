Overview

This is a small Todo application built with vanilla JavaScript. The app keeps its state(tasks + active filter) in memory and updates the DOM using a single update() function that re-renders the visible list.

JavaScript DOM Frontend

Features

  • Add task on Enter
  • Toggle complete/ active
  • Delete task
  • Filter: All/ Active/ Completed
  • Clear completed
  • Mark all as completed
  • Dynamic "items left" counter

Implementation idea

  • State is stored in an array: {text, completed}.
  • Any action modifies the array.
  • Then a single update() rebuilds the visible list and counter.
  • No incremental DOM patching, just clean re-render logic.

Representative code: filtering + re-render

function update() {
  lista.innerHTML = "";
  const vizibile = todos.filter(t =>
    filtru_curent === "all" ? true :
    filtru_curent === "active" ? !t.completed :
    t.completed
  );
  vizibile.forEach(t => {
    const li = document.createElement("li");
    lista.appendChild(li);
  });
  const ramase = todos.filter(t => !t.completed).length;
  count.textContent = `${ramase} items left`;
}

todo_nou.addEventListener("keypress", (e) => {
  if (e.key === "Enter" && todo_nou.value.trim()) {
    todos.push({ text: todo_nou.value.trim(), completed: false });
    todo_nou.value = "";
    update();
  }
});

The idea is simple: keep the data in memory, and every change triggers a full re-render. This keeps the logic predictable and easy to reason about.
