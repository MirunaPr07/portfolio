Overview

This project is a small arcade made in 2D OpenGL. It has two main parts: first, you build your own ship in an editor by placing blocks, and after that you use it in a Breakout-style game with bricks, a ball and a platform.

I focused on making the editor responsive (drag & drop, hover, click actions) and keeping the game logic readable: ball movement, collisions, brick removal and a simple start flow.

C++ OpenGL 2D Transforms Interaction

Main components

  • Ship editor - grid placement, drag & drop blocks, hover effect.
  • Constraints check - validates the ship before starting the game.
  • Game loop - updates ball motion and handles collisions.
  • Bricks - initialization + rendering + removal when hit a specific number of times.
  • UI state - switch between editor and game mode.

Key concepts used

  • 2D transformations (translate/scale/rotate) for composing the shapes.
  • Mouse interaction for editor actions (hover + click + drag).
  • Basic collision logic (ball vs bricks/ boundaries).
  • Keeping data in small structures or vectors for easy iteration.
  • Clear separation between rendering the editor and rendering the game.

Representative code: constraints validation

This is the part where I check the ship's configuration before allowing the player to start the game. I collect the placed blocks, separate normal solids from engines and verify the constraints stated. 

std::queue q;
std::vector> viz(linii, std::vector(coloane, 0));

q.push(blocuri[0]);
viz[blocuri[0].x][blocuri[0].y] = 1;

int cnt = 1;
while (!q.empty()) {
    glm::ivec2 crt = q.front();
    q.pop();

    const int dx[4] = { -1, 1, 0, 0 };
    const int dy[4] = { 0, 0, -1, 1 };

    for (int k = 0; k < 4; k++) {
        int nx = crt.x + dx[k];
        int ny = crt.y + dy[k];

        if (nx < 0 || ny < 0 || nx >= linii || ny >= coloane)
            continue;

        if (viz[nx][ny] == 1)
            continue;

        if (matrice[Index(nx, ny)].type == TipBloc::None)
            continue;

        viz[nx][ny] = 1;
        cnt++;
        q.push({ nx, ny });
    }
}
Short explanation

I keep the ship as a grid, then validate it with a couple of simple rules. After that, I run a BFS to check that the placed blocks are actually connected, not split.
