Overview

This project consists of a Turing Machine that validates a 4x4 Skyscrapers puzzle. The input contains both the grid and the visibility constraints, using digits and special symbols (1-4, :, #).

Instead of building one large machine from the start, I created separate Turing Machines for each validation, then combined them into a final version including all validations.

Turing Machine Formal Languages Theoretical CS

What I implemented

  • Row uniqueness validator.
  • Column uniqueness validator.
  • Visibility constraints validator.
  • Final machine integrating all constraints.

Implementation approach

  • Each rule was encoded as a separate state machine.
  • Transitions are defined explicitly in the .tms files.
  • The final machine implements all validations.
  • Acceptance state: Y; rejection state: N.

Representative snippet

Each transition is described by the current state and read symbol, followed by the next state, the symbol to write, and the direction. 

init : q0
accept : Y

q0, 1
q1, 1, >

q1, 2
q2, 2, >

q7, #
Y, #, -
Short explanation

The machine reads a symbol, transitions to a new state, optionally overwrites the symbol, and moves left, right, or stays in place. The computation ends in either an accept (Y) or reject (N) state.
