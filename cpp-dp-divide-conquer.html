Overview

This project contains four separate C++ programs. Each focuses on a different technique: dynamic programming (including careful update order), Divide & Conquer recursion, and a DP + BFS approach for optimization under constraints.

C++ Dynamic Programming Divide & Conquer BFS + Optimization

What is included

  • walsh.cpp - Divide & Conquer solution, without building the matrix.
  • stocks.cpp - 2D DP for maximizing profit with a loss constraint.
  • crypto.cpp - DP on a string with ? characters.
  • prinel.cpp - BFS for transformation costs + knapsack style DP for max gain.

How I approached it

  • I focused on clean definitions(especially for DP).
  • When needed, I used DP in reverse in order to avoid using the same step twice.
  • For Walsh, I reduced the problem to smaller sections recursively instead of building the entire matrix.
  • I kept the implementations straightforward and easy to test for each file.

Representative code: Divide & Conquer(from walsh.cpp)

This is the recursive function used to solve Walsh matrix problem by reducing the search to a smaller section. 

 bool divide_et_impera(int linie, int coloana, int m) {
    // linie si coloana le trimit functiei ca sa stie ce cauta

    // functia mea foloseste tehnica Divide Et Impera pentru ca mereu imi reduc
    // problema intreaga la o problema pe un sfert de matrice, impart problema
    // mereu in 4 si mi se reduce la o problema care tine strict de partea din
    // stanga sus a matricii
    if (m == 1)
        return 0;
    int mijloc;
    mijloc = m / 2;
    if (linie > mijloc && coloana > mijloc)
        return 1 - divide_et_impera(linie - mijloc, coloana - mijloc, mijloc);
    else {
        if (linie > mijloc)
            linie -= mijloc;
        if (coloana > mijloc)
            coloana -= mijloc;
        // aduc totul in stanga sus (toate celelalte in afara de dr jos sunt ac lucru)
        return divide_et_impera(linie, coloana, mijloc);
    }
}
Short explanation

I keep splitting the matrix into quadrants and map the requested pair of line and column into the corresponding subproblem. The only special case is the bottom-right quadrant, where the value is flipped.
